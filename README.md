# Final Project - 도서 쇼핑몰

## 담당 업무 : 메인페이지, 상품 리스트, 채팅 기능

+ 메인페이지 : 출간일 순으로 정렬한 책 카테고리별 출력 / 오늘의 책 출력 (book_id를 랜덤함수를 이용해 구한 다음 조회) / 베스트 셀러 촐력 (판매량 순)
+ 상품 리스트 : 카테고리별 상품 리스트 출력, 페이징 기능, 검색 기능
+ 채팅 기능 : 채팅하기 버튼을 클릭하면 채팅 팝업이 뜨며 채팅 서버에 접속하게 됨
           -> 1:1로 채팅 및 대화 내용 저장하는 기능을 만드려고 했으나 시간과 지식이 부족하여 접속 종료되면 대화 내용이 사라지고 모든 사람들이 접속되는 오픈 채팅방이 됨. 

## 주의 사항
+ 메인페이지의 경우 소스 그대로 사용하면 오류가 발생한다 -> 테이블 생성 후 각 카테고리별로 최소 4개의 데이터가 필요하며 order 테이블에도 최소 10개의 데이터가 필요하다
  발생 이유 : BookController에서 반복문을 사용해 데이터를 model에 저장하는 방식을 사용했기 때문에 Controller단에서 반복문을 실행하지 못해 오류가 발생한다.
+ 데이터를 넣지않고 사용할 경우 주소창에 확인할 페이지의 주소를 직접 입력해야 한다 -> 예시) localhost:8080/login (로그인 페이지로 이동)

## 사용 방법
+ 현 프로젝트는 InteliJ를 사용한 스프링 파일이며 sql과 연동이 필요하기 때문에 sql Id와 pw를 dispatcher-servlet.xml에서 수정이 필요하다.
+ 필요한 테이블은 ERD 파일을 참고하면 된다.
